The [Concepts](/docs/concepts) describe the parts. This section describes the
**system** — the two loops that make Canvasm an operating tool, not just a diagram:

```
   Strategy ──▶ Impact                Group ──▶ Dashboard
   actions/hypotheses                 audiences + access tags
        │                                   │
        ▼                                   ▼
   target / leading / guardrail        scoped operating views
   metrics, measured over a window     that respect visibility
```

## How they connect

Strategy-to-impact produces **impact badges** on dashboard metric widgets;
group-to-dashboard decides **which** widgets each audience sees. Together they mean a
team's review shows exactly the metrics they're allowed to see *and* the bets active
on them.
