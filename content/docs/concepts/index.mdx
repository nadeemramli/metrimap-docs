Canvasm (also called **Metrimap**) is built on a small set of ideas. Learn these
six and the rest of the product — guides, dashboards, connectors, the agent
API — will make sense.

## The model in one picture

```
Workspace  (your team's account)
└─ Canvas  (one metric map)
   ├─ Cards        outcomes, metrics, actions, hypotheses
   ├─ Relationships  typed links between cards
   ├─ Metrics      cards promoted to tracked, source-backed series
   ├─ Dashboard    widgets built from those metrics
   ├─ Strategy     actions/hypotheses with impact contracts
   └─ Visibility   groups + access tags decide who sees what
```

## A running example

Throughout these docs we model one business: a **subscription company** whose main
outcome is **monthly recurring revenue (MRR)**. MRR is driven by new, expansion,
and churned revenue; those are driven by acquisition and activation further down.
We'll grow that map as we go.
